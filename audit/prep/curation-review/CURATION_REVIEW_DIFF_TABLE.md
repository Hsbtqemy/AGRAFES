# Table différentielle — Curation : symptômes / correctifs
**Date** : 2026-03-13  
**Portée** : analyse code + audits existants (`CONTROL_MATRIX.md`, `dead_controls.md`)  
**Légende priorité** : 🔴 Bloquant éditorial | 🟠 Gêne significative | 🟡 Inconfort | 🟢 Amélioration souhaitable

---

| Zone | Symptôme observé | Impact utilisateur | Cause probable | Correctif minimal possible | Correctif plus ambitieux | Priorité |
|---|---|---|---|---|---|---|
| **Navigation doc** | Boutons Précédent/Suivant hard-disabled | Impossible de passer d'un document à l'autre sans revenir au select | `hard-disabled` HTML + absence de listeners dans `_renderCurationPanel()` malgré `_navigateCurateDoc()` implémentée | Attacher les listeners, activer/désactiver selon position dans `_docs[]` | Afficher un mini-queue visuel des 3 docs autour (réutiliser `_renderCurateQuickQueue()`) | 🔴 |
| **Navigation doc** | Label doc `#act-curate-doc-label` jamais alimenté | L'utilisateur ne sait pas quel doc est actif dans le header | `_refreshCurateHeaderState()` ne met pas à jour le label (dead selon `dead_controls.md`) | Appeler `_refreshCurateHeaderState()` après chaque changement de doc | Afficher titre + ID + statut workflow dans le label | 🟠 |
| **Preview — chips** | Chips `Brut` / `Diff` / `Côte à côte` / `+contexte` sans effet | Fausse affordance : l'utilisateur clique, rien ne se passe | `UI-only` dans `CONTROL_MATRIX.md` : aucun listener attaché | Supprimer les chips (pas de confusion) | Câbler `Brut` (masque diff, raw plein écran) et `Diff` (masque raw, diff plein écran) via toggle CSS `.preview-mode-*` | 🟠 |
| **Preview — sync** | Raw pane et diff pane scrollent indépendamment | Impossible de corréler une modification dans le diff avec sa position dans le texte source | Pas d'ancrage commun entre les deux panes ; pas d'état "modification active" | Aucun correctif minimal (nécessite `_activeDiffIndex`) | Scroll synchronisé piloté par `_activeDiffIndex` — `scrollIntoView()` dans chaque pane | 🔴 |
| **Preview — diff** | Aucune modification mise en avant après génération | L'utilisateur reçoit une liste plate sans point d'entrée | `_renderDiffList()` ne sélectionne pas la première entrée | Ajouter `.diff-item-first` CSS sur le premier item après render | Notion de modification active (`_activeDiffIndex = 0` par défaut après preview) | 🟡 |
| **Preview — diff** | Chaque ligne du diff n'indique pas quelle règle l'a générée | Impossible de savoir si une modification vient de `spaces`, `quotes` ou d'une regex custom | `_renderDiffList()` affiche `AVANT / APRÈS` sans attribut `rule_id` | Ajouter `data-rule="<id>"` si la réponse API le porte | Badge de règle visible sur chaque diff item, coloré par catégorie | 🟠 |
| **Preview — diff** | Pas de lignes de contexte (avant/après la modification) | La modification est lue hors contexte — impossible de juger si elle est correcte | `_renderDiffList()` ne récupère que les lignes modifiées, pas les lignes adjacentes | Ajouter ±1 ligne de contexte grâce aux données `examples` si disponibles | ±2–3 lignes de contexte, repliables, style grisé distinct | 🟠 |
| **Preview — minimap** | Minimap 12 buckets sans interaction | Vue densité utile mais on ne peut pas cliquer sur un bucket pour sauter à cette zone du diff | Minimap rendue par `_renderCurateMinimap()` sans listener de clic | Ajouter `cursor: pointer` + scroll du diff vers le bucket correspondant | Surligner le bucket actif selon `_activeDiffIndex` | 🟡 |
| **Pill mode** | `#act-curate-mode-pill` jamais synchronisé | L'utilisateur ne peut pas lire l'état du mode de curation dans le header | `dead` dans `dead_controls.md` : statique, jamais mis à jour | Alimenter la pill depuis `_refreshCurateHeaderState()` | Pill interactive : cliquer change le mode (document / sélection / corpus) | 🟡 |
| **Diagnostics — actionabilité** | Aucun item de diagnostic n'est cliquable | On sait que "guillemets : 14 applications" mais on ne peut pas les voir dans le diff | `_renderCurateDiag()` génère du HTML statique sans listener | Ajouter `cursor: pointer` + filtre `data-rule` sur `#act-diff-list` | Diagnostics devient un panneau de filtrage avec badge "actif" et compteur "N filtrés" | 🔴 |
| **Diagnostics — densité** | Pas de scroll interne, contenu masqué si trop d'entrées | Règles nombreuses ou libellés longs tronqués ou invisibles | Colonne droite 300px sans `overflow-y: auto` sur le conteneur interne | Ajouter `overflow-y: auto; max-height: calc(100% - Xpx)` sur `#act-curate-diag` | Colonnes redimensionnables ou colonne droite plus large (voir 1.5) | 🟠 |
| **Diagnostics — structure** | Résumé global (N modif, N règles) mélangé avec le détail | Impossible de garder un œil sur les totaux tout en faisant défiler le détail | Pas de séparation entre "résumé" et "liste de règles" dans `_renderCurateDiag()` | Séparer en deux `<div>` : résumé sticky + liste scrollable | Résumé sticky avec fond distinct, règles filtrables, journal replié par défaut | 🟠 |
| **Journal de revue** | Max 10 entrées, pas de persistance, non interactif | Trace d'actions trop courte, disparaît à la preview suivante | `_pushCurateLog()` limite à 10, `_curateLog[]` en mémoire volatile | Augmenter la limite à 50, ajouter scroll | Journal persistant dans la session + export optionnel (texte) | 🟡 |
| **Correction locale — granularité** | L'unité d'action minimale est le document entier | Impossible d'accepter certaines modifications et d'en refuser d'autres | Pas d'objet "diff item" en mémoire avec état ; `_runCurate()` opère en bloc | Aucun correctif minimal sans restructuration de l'état | Introduire `_diffs: DiffItem[]` avec `status: "pending" | "accepted" | "ignored"` | 🔴 |
| **Correction locale — actions** | Aucun bouton Accepter / Ignorer / Éditer par modification | Travail éditorial fin impossible — tout ou rien | Pas de granularité d'action ; `enqueueJob("curate")` opère sur tout le document | Aucun correctif minimal | Actions locales par `.diff-item` : `⌫ Ignorer` (niveau 2) puis `✎ Éditer` (niveau 3) | 🔴 |
| **File de revue** | `#act-curate-queue` toujours vide (placeholder dead) | La file de revue "3 docs autour du courant" n'est jamais rendue | `_renderCurateQuickQueue()` existe mais n'est probablement pas appelée au bon moment | Appeler `_renderCurateQuickQueue()` après chaque navigation doc | Intégrer la file dans la colonne gauche avec indicateur de progression (X/N revus) | 🟠 |
| **Colonne droite — largeur** | 300px — labels de règles regex tronqués | Règles complexes illisibles dans Diagnostics | Grille CSS `310px minmax(580px,1fr) 300px` — droite fixe | `320px` ou `340px` | Colonne droite redimensionnable par drag | 🟡 |
| **État pill mode** | Pill mode statique "prévisualisation" | L'utilisateur croit toujours être en mode preview même après avoir appliqué | `_refreshCurateHeaderState()` non appelée après `_runCurate()` | Appeler la méthode après chaque action principale | Mode dynamique reflétant l'état réel (preview / appliqué / modifié localement) | 🟡 |
