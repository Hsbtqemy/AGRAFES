# TEI Profile — multicorpus_engine

**Version:** 1.0 (2026-03-01)
**Status:** Accepted

This document describes the TEI XML profile used by `multicorpus_engine` for
import and export. It defines which TEI tags are supported, how internal fields
map to TEI constructs, and the guarantees provided by each export level.

---

## 1. Supported Tags at Import (`tei_importer`)

The TEI importer reads `<p>` or `<s>` elements as **line units** (configurable
via `unit_element`). Namespace-aware: handles `xmlns="http://www.tei-c.org/ns/1.0"`
and no-namespace TEI.

| Tag | Behaviour | Stored as |
|-----|-----------|-----------|
| `<p xml:id="p1">` | Line unit, external_id from trailing int of xml:id | `unit_type=line, external_id=1` |
| `<s xml:id="s42">` | Line unit (if unit_element=s), external_id from xml:id | `unit_type=line, external_id=42` |
| `<head>` | **Not imported** (structure units come from TXT/DOCX importers) | — |
| `<div>` | Container, traversed but not stored | — |
| `xml:id` without trailing int | Sequential `n` used as external_id fallback | `external_id=n` |

**Title resolution:** `teiHeader//title[0]` → fallback to filename stem.  
**Language resolution:** `xml:lang` on `<text>` or `<TEI>` → overridden by `--language`.

## 2. teiHeader Fields Guaranteed at Export

All exported TEI files contain a `<teiHeader>` with the following structure:

```xml
<teiHeader>
  <fileDesc>
    <titleStmt>
      <title><!-- documents.title --></title>
      <!-- Sprint 3+: author, date if in meta_json -->
    </titleStmt>
    <publicationStmt>
      <p>Generated by multicorpus_engine on <!-- created_at --></p>
    </publicationStmt>
    <sourceDesc>
      <p><!-- documents.source_path or "unknown" --></p>
    </sourceDesc>
  </fileDesc>
  <profileDesc>
    <langUsage>
      <language ident="<!-- documents.language -->">
        <!-- documents.language -->
      </language>
    </langUsage>
    <!-- Sprint 3+: textClass with doc_role / resource_type -->
    <!-- Sprint 3+: listRelation for translation_of / excerpt_of -->
  </profileDesc>
</teiHeader>
```

## 3. ID Strategy

Units are identified by two complementary attributes:

| Attribute | Value | Purpose |
|-----------|-------|---------|
| `xml:id` | `u{unit_id}` (line) / `s{unit_id}` (structure) | Stable cross-document reference |
| `n` | `external_id` if set, else sequential `n` | Human-readable sequence number |

**Example:**
```xml
<p xml:id="u42" n="3">Text of the unit.</p>
```

---

## 4. Export Levels

### 4a. Analysis level (default, `include_structure=False`)

Exports only **line units** (`unit_type=line`) as `<p>` elements.
Suitable for computational analysis and alignment.

### 4b. Structural level (`include_structure=True`)

Adds **structure units** (`unit_type=structure`) as `<head>` elements,
preserving the document's structural outline.

### 4c. With Alignments (`include_alignment=True`, Sprint 2+)

Adds a `<linkGrp type="alignment">` section in `<text>` after `<body>`:

```xml
<linkGrp type="alignment" corresp="doc_{pivot_id}.tei.xml">
  <link target="#u{pivot_unit_id} #u{target_unit_id}"/>
  <!-- one <link> per alignment link -->
</linkGrp>
```

Status filter: `status_filter` (default `["accepted"]`) — export only accepted
links; set to `["accepted","unreviewed"]` to include unreviewed.

---

## 5. Relation Encoding (Sprint 3+)

Document-level relations (from `doc_relations` table) are encoded in `profileDesc`:

```xml
<profileDesc>
  ...
  <listRelation>
    <relation type="translation_of" active="#this" passive="#doc_2"/>
  </listRelation>
</profileDesc>
```

Relation types supported: `translation_of`, `excerpt_of`.

---

## 6. Publication Package (Sprint 4+)

A ZIP archive `agrafes_publication_{date}.zip` containing:
- `tei/*.xml` — one file per document
- `manifest.json` — document list, alignments count, export parameters
- `checksums.txt` — SHA-256 per file
- `README.md` — human-readable summary

---

## 7. Warnings and Validation

Missing required fields emit structured warnings in `stats_json.warnings`:

```json
{"type": "tei_missing_field", "field": "title", "doc_id": 3}
```

These warnings do **not** prevent export; they flag data quality issues.
