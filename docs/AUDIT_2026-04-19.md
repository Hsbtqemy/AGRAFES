## 2. Findings

### 🔴 Bloquant

Ces points peuvent provoquer une corruption de données, un crash, ou une faille exploitable.

| # | Fichier | Ligne | Problème |
|---|---------|-------|----------|
| B1 | `tauri-shell/src/shell.ts` | `_switchDb` | **Race condition** — appels concurrents possibles si l'utilisateur clique rapidement sur « Ouvrir DB » ; deux sidecars peuvent démarrer simultanément sur le même portfile |
| B2 | `tauri-prep/src/screens/AnnotationView.ts` | boucle pagination | **Boucle après dispose()** — si l'utilisateur change de vue pendant le chargement des tokens, la boucle `while(true)` continue à écrire dans `this._annotTokens` et à appeler `this._panel` → crash (`null` deref) |
| B3 | `db/migrations.py` | `apply_migrations()` | **Pas de verrou** — deux appels concurrents (ex. sidecar multi-thread) peuvent tenter d'insérer le même numéro de migration → `PRIMARY KEY violation` |
| B4 | `db/migrations.py` | `executescript()` | **Sans transaction explicite** — une coupure pendant une migration laisse le schéma dans un état partiellement appliqué, non récupérable |
| B5 | `exporters/conllu_export.py` | `_conllu_field()` | **Tabulation non échappée** — si `word` ou `lemma` contient un `\t`, le fichier CoNLL-U exporté est corrompu (format colonne-séparées-par-tab) |
| B6 | `importers/*.py` (conllu, docx, txt) | import handlers | **Pas de rollback** — en cas d'erreur partielle, les unités déjà insérées restent en base → documents orphelins non indexés, incohérence FTS |
| B7 | `tauri-prep/src/screens/ActionsScreen.ts` | ~591 | **XSS DOM** — `tr.innerHTML` construit avec `doc.title`, `doc.language`, `doc.doc_role` non échappés ; un titre de document comme `<img src=x onerror="…">` s'exécute dans le WebView |

## 3. Zones sans finding

Ces modules ont été lus sans problème notable identifié :

- `exporters/` : csv_align, html, jsonl, ske, readable_text, tmx, tei_package (escaping correct partout)
- `aligner.py` : requêtes paramétrées, pas d'O(n²) sans borne
- `segmenter.py` : regex sans risque ReDoS, pas de boucle infinie
- `query.py` + `token_query.py` : pas d'injection FTS5/SQL, bornage des répétitions en CQL
- `scripts/build_sidecar.py` + `sidecar_entry.py` : `subprocess.run()` via liste argv, pas de `shell=True`
- `tauri-app/src/lib/sidecarClient.ts` : token jamais loggé ni dans les URLs

## 4. Priorisation recommandée

### Traiter en premier (risque données / sécurité)

1. **B7** — XSS ActionsScreen : remplacer `innerHTML` par `textContent` ou utiliser `_escHtml()`
2. **B5** — CoNLL-U tab escaping : `_conllu_field()` doit remplacer `\t` par espace ou lever une erreur
3. **B3 / B4** — migrations : ajouter un verrou (`threading.Lock`) + wrapper `try/except` avec rollback explicite
4. **B2** — AnnotationView dispose guard : ajouter `if (!this._panel) return` en début de boucle
5. **B6** — Importers rollback : encapsuler l'import dans une transaction SQLite unique

### Traiter ensuite (stabilité long terme)

6. **D7 / D8** — XML importers : remplacer par `defusedxml.ElementTree`
7. **D2 / D3 / D4 / D5** — fuites listeners/Chart.js : cleanup systématique dans `dispose()`
8. **D11** — JobManager TTL : purger les jobs terminés après N minutes
9. **D9** — Importers : vérifier la taille du fichier avant lecture (`os.path.getsize`)
10. **D6** — TEI exporter : valider `output_path` avec `Path.resolve()` + assert dans répertoire autorisé

### Traiter quand opportun

11. **B1** — `_switchDb` : ajouter un flag `_switching` pour sérialiser les appels
12. **C1 / C2 / C3 / C4** — cosmétiques

## 5. Bilan général

Le projet est **fonctionnellement solide** : API correctement versionnée, contrat de snapshot, requêtes paramétrées, pas d'injection SQL identifiée. Les failles trouvées sont classiques d'un projet passé rapidement de prototype à application distribuée : gestion du cycle de vie des composants UI (listeners, instances graphiques) et robustesse des couches I/O (transactions, taille fichier, entités XML).

L'issue la plus urgente est le **XSS en B7** : dans une application de bureau Tauri, l'impact est limité (pas d'exfiltration réseau vers un attaquant externe), mais un corpus importé depuis une source tierce pourrait déclencher du code arbitraire dans le WebView.
